# 宝宝辅食排敏记录 - 产品设计方案

> 版本: v1.0 | 更新日期: 2025-06-25

---

## 1. 产品概述

### 1.1 产品定位
一款面向 0-3 岁宝宝家长的**手机端 Web 应用**（PWA），帮助家长系统化记录宝宝的辅食添加和排敏过程，追踪过敏源，管理饮食健康。

### 1.2 核心功能
| 功能模块 | 说明 |
|---------|------|
| 宝宝档案 | 创建/编辑宝宝基本信息（昵称、生日） |
| 食物排敏管理 | 预置常见食物分类清单，记录每种食物的排敏状态 |
| 辅食记录 | 按日期+餐次记录宝宝吃过的食物及过敏反应 |
| 日历视图 | 月历模式查看每日辅食记录概览 |
| 食物总览 | 查看所有食物的排敏状态和食用频次 |
| 数据导出 | 将记录导出为 Excel 文件供下载 |

### 1.3 设计风格
- **色调**：米黄色 + 浅色系为主，温馨明亮
  - 主色：`#F5E6D3`（米黄）
  - 辅色：`#FF9E6D`（暖橙）
  - 成功色：`#7BC67E`（柔绿 - 不过敏）
  - 警告色：`#FFB347`（暖黄 - 观察中）
  - 危险色：`#FF6B6B`（珊瑚红 - 过敏）
  - 背景色：`#FFF8F0`（奶油白）
  - 卡片色：`#FFFFFF`（纯白）
  - 文字主色：`#4A3728`（深棕）
  - 文字次色：`#8B7355`（浅棕）
- **布局**：竖屏优先，大按钮/大字体，便于单手操作
- **圆角**：全局 12px-16px 圆角，柔和可爱
- **阴影**：轻投影，增加层次感但不压迫

### 2.3 数据流转
```

```
- **配置数据**（食物清单）：从 `foodConfig.ts` 读取，不可修改
- **用户数据**（宝宝信息、记录）：存储在 LocalStorage，支持 CRUD

## 3. 数据模型

### 3.1 宝宝信息 (BabyProfile)
```typescript
interface BabyProfile {
  id: string;               // UUID
  name: string;             // 宝宝昵称
  birthday: string;         // 生日 ISO 格式：YYYY-MM-DD
  avatar?: string;          // 头像（预留）
  createdAt: string;        // 创建时间
  updatedAt: string;        // 更新时间
}
```

### 3.2 食物记录 (FoodRecord)
```typescript
interface FoodRecord {
  id: string;               // UUID
  date: string;             // 日期 YYYY-MM-DD
  meal: MealType;           // 餐次：breakfast | lunch | snack | dinner
  foodId: string;           // 食物 ID（对应配置表）
  foodName: string;         // 食物名称（冗余存储，便于展示）
  reaction: ReactionType;   // 排敏反应：safe | observing | allergic
  dayCount: DayCount;       // 排敏天数：day1 | day2 | day3
  note: string;             // 备注
  createdAt: string;        // 创建时间
}
```

### 3.3 食物配置 (FoodConfig)
```typescript
interface FoodCategory {
  id: string;               // 分类 ID
  name: string;             // 分类名称
  icon: string;             // emoji 图标
  foods: FoodItem[];        // 该分类下的所有食物
}

interface FoodItem {
  id: string;               // 食物唯一 ID
  name: string;             // 食物名称
  allergenLevel: 'low' | 'medium' | 'high';  // 致敏风险等级
  recommendedAge: string;   // 建议月龄，如 "6+" 表示 6月龄以上
}
```

### 3.4 枚举定义
```typescript
type MealType = 'breakfast' | 'lunch' | 'snack' | 'dinner';
type ReactionType = 'safe' | 'observing' | 'allergic';
type DayCount = 'day1' | 'day2' | 'day3';

// 中文映射
const MEAL_LABELS: Record<MealType, string> = {
  breakfast: '早餐', lunch: '午餐', snack: '下午加餐', dinner: '晚餐'
};
const REACTION_LABELS: Record<ReactionType, string> = {
  safe: '不过敏', observing: '观察中', allergic: '过敏'
};
const DAY_LABELS: Record<DayCount, string> = {
  day1: '第一天', day2: '第二天', day3: '第三天'
};
```

### 3.5 LocalStorage 键设计
| Key | 类型 | 说明 |
|-----|------|------|
| `baby_profile` | `BabyProfile` | 宝宝信息（单个） |
| `food_records` | `FoodRecord[]` | 所有辅食记录 |
| `preset_allergens` | `string[]` | 初始化时已排敏的食物 ID 列表 |

## 6. 核心交互流程

### 6.1 首次使用流程
```
打开 App
  ├── 有宝宝数据？ → 进入主页面（首页 Tab）
  └── 无宝宝数据？ → 进入引导页
                        ├── Step1: 创建宝宝（昵称+生日）
                        ├── Step2: 选择已排敏食物
                        └── 保存 → 进入主页面
```

### 6.2 日常使用流程
```
打开 App → 首页
  ├── 查看今日统计
  ├── 查看本周日历 → 点击某天 → 查看该天记录
  ├── 点击"添加记录" → 打开记录面板 → 填写 → 保存
  └── 点击食物分类 → 查看分类详情
```

### 6.3 排敏记录流程（标准三天排敏法）
```
Day 1: 添加新食物，选择"第一天"，排敏反应选"观察中"
Day 2: 再次添加该食物，选择"第二天"，根据情况选排敏反应
Day 3: 第三次添加，选择"第三天"，最终确认排敏结果
  ├── 三天均无反应 → 标记为"不过敏/安全"
  ├── 有轻微反应 → 标记为"观察中"，暂停该食物
  └── 明确过敏 → 标记为"过敏"
```
